01 — Privilèges système & escalade de droits01 — System privileges & privilege escalation01 — システム権限とエスカレーション

La gestion des privilèges sur les systèmes Linux reste l'un des vecteurs d'attaque les plus exploités. La faille CVE-2021-3156 « Baron Samedit » dans sudo, restée active en production sur des systèmes non mis à jour, illustre ce phénomène : un utilisateur local peut, par un débordement de tampon (heap overflow), obtenir les droits root sans aucun mot de passe. Cette vulnérabilité, corrigée en version 1.9.5p2, a mis en évidence l'importance critique du patch management et du principe de moindre privilège. Dans mes travaux pratiques GSB (Mission 3), j'ai directement manipulé les politiques de groupe Active Directory et les scripts PowerShell pour enforcer ce principe dans un environnement Windows Server hybride. Linux privilege management remains one of the most exploited attack vectors. The CVE-2021-3156 "Baron Samedit" flaw in sudo, still active on unpatched systems, illustrates this: a local user can, via heap overflow, gain root rights without any password. This vulnerability, fixed in version 1.9.5p2, highlighted the critical importance of patch management and the principle of least privilege. In my GSB lab work (Mission 3), I directly manipulated Active Directory group policies and PowerShell scripts to enforce this principle in a hybrid Windows Server environment. Linuxシステムでの権限管理は最も悪用される攻撃ベクターの一つである。sudoのCVE-2021-3156 "Baron Samedit"脆弱性は、未パッチシステムでローカルユーザーがパスワードなしにroot権限を取得可能にする。バージョン1.9.5p2で修正されたこの脆弱性は、パッチ管理と最小権限の原則の重要性を示した。

Lors de mon stage EnerSys (2025–2026), j'ai constaté que plusieurs postes Linux maintenus en production tournaient sur des versions de sudo antérieures aux correctifs. La mise à jour systématique faisait partie des recommandations que j'ai formulées dans mes documentations techniques. During my EnerSys internship (2025–2026), I observed that several Linux production workstations ran pre-patch versions of sudo. 02 — Segmentation réseau : VLANs, Zero Trust & menaces internes02 — Network segmentation: VLANs, Zero Trust & insider threats02 — ネットワーク分割：VLAN、ゼロトラスト、内部脅威

La segmentation réseau a profondément évolué ces deux dernières années. Le modèle VLAN 802.1q demeure la référence en PME et en environnement industriel — ce que j'ai pu vérifier directement chez EnerSys avec la configuration Cisco Meraki — mais l'émergence du cloud hybride rend ce modèle insuffisant pour un Zero Trust réel. Les attaques de type VLAN hopping, qui exploitent les mauvaises configurations des ports trunk (mode DTP auto laissé actif), permettent à un attaquant positionné en interne de traverser les segments réseau sans autorisation. Ces techniques sont documentées dans le framework MITRE ATT&CK (T1599). La micro-segmentation, implémentée via des solutions NGFWs ou des SDN, constitue la réponse architecturale adaptée aux environnements hybrides. Network segmentation has evolved significantly over the past two years. The VLAN 802.1q model remains the SME and industrial standard — verified directly at EnerSys with Cisco Meraki — but hybrid cloud growth makes this model insufficient for true Zero Trust. VLAN hopping attacks, exploiting trunk port misconfigurations (active DTP auto mode), allow internally-positioned attackers to traverse network segments without authorization. These techniques are documented in the MITRE ATT&CK framework (T1599). Micro-segmentation via NGFWs or SDN is the appropriate architectural response for hybrid environments. VLAN 802.1qはSMEおよび産業環境の標準であるが、ハイブリッドクラウドでは真のゼロトラストとして不十分である。VLAN hopping攻撃はトランクポートの誤設定を悪用し、内部攻撃者が認可なしにセグメントを越えることを可能にする。NGFWやSDNによるマイクロセグメンテーションが適切な対策である。

03 — IA en cyberdéfense : SIEM nouvelle génération & détection comportementale03 — AI in cyberdefense: next-gen SIEM & behavioral detection03 — サイバー防衛のAI：次世代SIEM & 行動検知

L'intégration de l'intelligence artificielle dans les outils de détection représente la transformation la plus significative de cette période. Les SIEM de nouvelle génération intègrent désormais des modules UEBA (User and Entity Behavior Analytics) basés sur le machine learning, permettant de détecter des anomalies comportementales que les règles statiques ne peuvent pas identifier : un compte administrateur se connectant à 3h du matin depuis une IP étrangère, ou un transfert de données anormalement volumineux vers un stockage cloud externe. Ces systèmes réduisent drastiquement les faux positifs — jusqu'à 60% selon les benchmarks Gartner 2024 — et permettent la corrélation en temps réel de milliers d'événements pour identifier des APT (Advanced Persistent Threats) dont la furtivité repose précisément sur l'étalement de l'attaque dans le temps. AI integration in detection tools represents the most significant transformation of this period. Next-gen SIEMs now include machine learning-based UEBA (User and Entity Behavior Analytics) modules, detecting behavioral anomalies that static rules cannot identify: an admin account connecting at 3am from a foreign IP, or abnormally large data transfers to external cloud storage. These systems drastically reduce false positives — up to 60% according to 2024 Gartner benchmarks — and enable real-time correlation of thousands of events to identify APTs whose stealth relies on spreading the attack over time. AIの検知ツールへの統合はこの期間で最も重要な変革である。次世代SIEMにはMLベースのUEBAモジュールが搭載され、静的ルールでは検出不能な行動異常を検知できる。誤検知を最大60%（Gartner 2024比）削減し、時間をかけて潜伏するAPTをリアルタイムで相関検出する。

My Discord cybersecurity monitoring bot (Python + PostgreSQL + systemd) directly implements this automated correlation principle: it aggregates CVE, CERT-FR and ANSSI feeds to redistribute critical alerts in real-time, reproducing at a smaller scale the logic of a lightweight SIEM. 私のDiscordサイバーセキュリティ監視ボット（Python + PostgreSQL + systemd）はこの自動相関原則を直接実装している：CVE、CERT-FR、ANSSIフィードを集約してリアルタイムで重要アラートを再配信し、軽量SIEMのロジックを小規模で再現する。

04 — Chiffrement et gestion des endpoints : le cas BitLocker / Intune04 — Encryption & endpoint management: the BitLocker / Intune case04 — 暗号化とエンドポイント管理：BitLocker / Intuneの事例

La migration des solutions de chiffrement full-disk vers des outils natifs Microsoft — illustrée par la migration Trellix → BitLocker que j'ai conduite chez EnerSys — reflète une tendance de fond : la consolidation des stacks de sécurité autour des écosystements existants (Microsoft 365 / Intune).
